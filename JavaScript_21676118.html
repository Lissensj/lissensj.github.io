Using The API

During the creation of a Qeo reader or writer you can optionally enable fine grained access control policy handling and specify a policyUpdate event handler.  If the event handler is specified then Qeo will call it:

  1. When creating the reader or writer,
  2. Whenever a Qeo Policy update is received,
  3. When requesting a Qeo Policy re-evaluation (i.e. make the app-based policy effective towards Qeo).

When the event handler gets called:

  1. It will receive a Qeo Policy JSON object, containing the users (identities) to whom you're allowed to communicate in your realm. This is defined by the policy of the realm.

When we want to further restrict communication for a certain reader or writer:

  1. We should decide for all users (identities) of the Qeo Policy JSON object whether we allow or deny the users from being read from or written to.
  2. We should trigger the policy update method of the involved reader or writer to request a re-evaluation of the Qeo Policy.
    When calling this method the policyUpdate event handler will be called again to know the result of your request.

 

Enabling  fine grained access control policy handling during reader/writer creation

factory.createStateWriter("org::qeo::junit::State", {"enablePolicy": true, "on": {"policyUpdate": policyUpdateWriter}});
factory.createStateReader("org::qeo::junit::State", {"enablePolicy": true, "on": {"policyUpdate": policyUpdateReader}});

The policyUpdate event handlers

var policyWriter, policyReader;

function policyUpdateWriter(_policy) {
	 policyWriter = _policy;
 	 console.log("policyUpdate for writer" + JSON.stringify(policyWriter));
}

function policyUpdateReader(policy) {
	policyReader = _policy;
 	console.log("policyUpdate for reader" + JSON.stringify(policyReader));
}

Requesting a re-evaluation of the Qeo Policy

We trigger a re-evaluation of the Qeo Policy when we call the updatePolicy method of a reader or writer with as argument the updated Qeo JSON Policy object.
When calling this method the policyUpdate event handler for the reader/writer will be called again to know the result of your request.

writer.updatePolicy(policyWriter);
reader.updatePolicy(policyReader);

 

Requesting the currently enfored fine grained access control policy

writer.requestPolicy();
reader.requestPolicy();
