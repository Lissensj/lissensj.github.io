Qeo Native Documentation : QDM Catalog

Created by lissensj on May 21, 2014

About The QDM Catalog

The Qeo SDK ships with some predefined Qeo Data Models.  These QDMs are used by the Qeo framework and some of the samples.  They can also be used when developing new applications.

Catalog Contents

In the qdms/qdm folder of the SDK you will find a.o. a catalog folder.  This folder contains the predefined Qeo Data Models mentioned above.

  • qeo_types.xml: Contains some predefined types that can be used in other QDMs.
  • qeo_DeviceInfo.xml: Contains the device information type published by Qeo (see also QFinder App Description).

Catalog Generated Code

In the qdms/qdm folder of the SDK you will find subfolders with generated code for the catalog QDMs.

Android

No folder is present for Android since the generated code is already part of the Android Qeo library.  You can automatically use the predefined QDMs.

C

The c folder contains the generated C header files for the catalog QDMs.  The compiled code corresponding to these header files is part of the Qeo library.  The header files can be included in new applications to also make use of these predefined QDMs.

Icon

Do not generate C code for the predefined QDMs and link these into your own application since this might result in duplicate symbol warnings or errors from the linker.  Instead use the ones that are already present in the Qeo library.

Javascript

The js folder contains the generated JavaScript files for the catalog QDMs.  New applications can make use of these predefined QDMs by loading them using the HTML script tag.
